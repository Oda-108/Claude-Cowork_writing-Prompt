# X（Twitter）バズ記事作成スキル

## このスキルの目的

X（Twitter）のArticles機能を使った長文記事を、バズの構造に基づいて作成する。
実際に337万インプレッション（いいね2,679）、137万インプレッション（いいね508）、63万インプレッション（いいね1,019）を叩き出した記事群を分析して抽出した法則をすべて組み込む。

「なんとなく良い記事」ではなく、**読者の心理を構造的に設計し、最後のCTAまで離脱させない記事**を生成する。

---

## 記事作成の前提：必ずユーザーに確認する項目

記事を書き始める前に、以下を確認すること。不明な場合はユーザーに質問する。

1. **テーマ** — 何について書くか（例：AIスキル、心理学、キャリア、ツール紹介）
2. **ターゲット読者** — 誰に読ませたいか（例：20代ビジネスマン、副業初心者、経営者）
3. **文体タイプ** — 以下の3タイプから選択
   - **攻撃型**（@proneeeet風）：タメ口・煽り・命令形。「〜しろ」「〜な奴は終わってる」
   - **知的メンター型**（@fudoosanya風）：丁寧語ベース。「〜ですよね」「〜なんです」知的で親しみやすい
   - **体験共有型**（@oda_nobunaga10風）：「〜した」「〜だった」の過去形。友達が興奮して話す感じ
4. **CTA先** — 読者を最終的にどこへ誘導するか（フォロー、LINE、コミュニティ、商品ページなど）
5. **画像の方向性** — どんな画像を使うか（マンガ風、ダーク系、スクリーンショット、ミニマルなど）
6. **リスト数** — 本文のセクション数（推奨：3〜7つ。5が黄金比）

## 記事構造テンプレート（9パート）

以下の順序で記事を構成する。各パートの役割と書き方を厳密に守ること。

### パート1：タイトル

タイトルは記事のクリック率の80%を決める。以下の要素を組み合わせて構築する。

**必須要素（最低2つ組み合わせる）：**

| 要素 | 効果 | 例 |
|------|------|-----|
| 【隅付き括弧ラベル】 | TLで視覚的に目立つ。禁忌感・カテゴリ明示 | 【禁断】【生存戦略】【暴露】【警告】 |
| 「鉤括弧でキャッチフレーズ」 | 読者の言葉 or 衝撃ワードを際立たせる | 「とんでもない金持ち」「AIすげぇw」 |
| 数字 | 具体性＋読了の見通し | 7つ、5選、14タスク、3ステップ |
| 読者の欲望/恐怖を直撃する結果 | ベネフィットを明示 | 「成り上がる」「洗脳レベルで説得」「給料の上げ方」 |
| 読者への呼びかけ/挑発 | 自尊心を刺激し「自分は違う」と証明するために読ませる | 「情弱へ」「知らない奴は搾取される」 |

**タイトルパターン（実績あり）：**

- パターンA：【禁忌ラベル】+ 時期 +「強烈な結果」+ 数字リスト
  → 例：【禁断】2026年に「とんでもない金持ち」に成り上がる7つのAIスキル

- パターンB：【カテゴリラベル】+「読者の行動を皮肉で引用」+ 呼びかけ
  → 例：【生存戦略】「AIすげぇw」でハシャいでる情弱へ

- パターンC：結果 + 手段 + 数字リスト
  → 例：相手を洗脳レベルで説得する心理学5選

- パターンD：超シンプル＋普遍的欲求
  → 例：給料の上げ方

- パターンE：ツール名 + 具体数字 + ストーリー予告
  → 例：Claude Coworkで1日14タスクを処理した日の話をする

**タイトルのNG：**
- 抽象的すぎる（「AIの未来について」→ 何が得られるか不明）
- 長すぎる（40文字以上はTLで切れる）
- 結果が不明（「考えてみた」→ 読む理由がない）

---

### パート2：ヘッダー画像（アイキャッチ）

タイムラインで最初に目に入る視覚要素。画像の方向性は文体タイプと一致させる。

| 文体タイプ | 推奨画像スタイル | 効果 |
|-----------|----------------|------|
| 攻撃型 | マンガ風キャラ（不敵な笑み、ダーク系） | 「闇の知識」感。サブカル層に刺さる |
| 知的メンター型 | ダーク背景＋白文字のみ（ミニマル） | 知的・洗練された印象。TLでの視覚的アイデンティティ |
| 体験共有型 | 実際の画面スクリーンショット | 「証拠」として機能。信頼性UP |

**画像の鉄則：**
- ダークモードのTLで映える色調（黒背景＋白/金文字が最強）
- ブランド統一（毎回同じテイストにして、TLで「あ、この人だ」と認識させる）
- タイトル文字は画像内にも大きく入れる（画像だけで内容が分かるようにする）

---

### パート3：フック（冒頭3行）

記事を開いた読者が最初に読む3行。ここで離脱するか読み続けるかが決まる。

**フックの4パターン：**

**①恐怖トリガー型（攻撃型向き）**
読者の現状を引用して否定し、恐怖を煽る。
```
「AIで副業！」「Claude Codeで自動化w」
そんなレベルでキャッキャしてる奴ら、全員まとめて葬り去られる準備はできてるか？
```
構造：読者の言葉を引用 → 否定 → 恐怖の疑問形

**②共感トリガー型（知的メンター型向き）**
読者の内面のモノローグを再現する。
```
「なんであいつ、あんなに説得力あるんだろ？」
「自分の言うことだけ、全然通らないな……」
そう思うこと、ありませんか。
```
構造：内面の声を再現 → 「ありませんか」で問いかけ

**③欲望トリガー型**
富・成功・効率などの欲望に直接訴える。
```
今の時代、「AIスキル格差」は過去20年で最大の富のボーナスタイムだ。
世の中の9割はAIをただの検索の延長にしか使ってない。
だが、ごく一部の「勝ち組」は、AIを使って24時間不眠不休で札束を刷る方法に気づいちまった。
```
構造：大きな主張 → 格差の提示（9割 vs ごく一部）→ 具体的なベネフィット

**④事実提示型（体験共有型向き）**
驚くべき事実をストレートに提示する。
```
正直に言う。昨日、Claude Coworkで1日に14タスクをこなした。
```
構造：宣言 → 衝撃的な事実 → 具体的な列挙

**フックのチェックリスト：**
- [ ] 3行以内で感情が動くか？（恐怖/共感/好奇心/欲望）
- [ ] 読者が「自分のことだ」と感じるか？
- [ ] 「続きを読まないと損」と思わせるか？
- [ ] 疑問形 or 常識否定で「答えを知りたい」欲を刺激しているか？

### パート4：問題の提起（6教育①②③の組み込み）

フックの直後で、読者の「現在の世界観」を壊す。ここで3つの教育を同時に行う。

**① 目的の教育**（なぜこの記事を読む必要があるのか）
- 「知っているか、知らないか。それだけで、人生の難易度が変わります」
- 「搾取される側から抜け出すための知識」
- 「この記事を読んだ後、あなたの行動が変わる」

**② 信頼の教育**（なぜ自分の話を聞くべきか）
- 実績の提示：「利益率80%で〜」「1日14タスクを処理した」
- 内部者の視点：「AIの最前線で地ならしを感じてきた」
- 権威の借用：「西野亮廣さんの思想をベースに」「Googleの元エンジニアが絶賛」
- スクリーンショットや数字の証拠

**③ 問題の教育**（読者の何が間違っているか）
- 「頑張れば給料が上がるなんて幻想」
- 「1つのAIで満足してる奴は一生三流」
- 「無料版で満足してる層という名の養分」

**書き方の鉄則：**
- 読者の「常識」を1つだけ選び、それを明確に否定する
- 否定した後、すぐに「正しい答えがある」ことを匂わせる（本文への橋渡し）
- 自分の権威を1〜2文で確立する（長すぎると自慢に聞こえる）

---

### パート5：本文（リスト型セクション）— 6教育④の組み込み

記事の本体。ここが「手段の教育」（どうすれば解決できるか）。

**構造ルール：**

1. **3〜7つのナンバリングされたセクション**（推奨5つ）
2. 各セクションは以下の構造：
   ```
   見出し（太字 or 番号付き）
   ├ 主張（1〜2文。これがこのセクションの結論）
   ├ 具体例 or アナロジー（「例えるなら〜」「具体的には〜」）
   ├ 数字 or エビデンス（「〜万円」「〜%」「〜ヶ月ごとに倍増」）
   ├ 画像（証拠 or 感情増幅。テキストだけで伝わりにくい概念を可視化）
   └ 読者への問いかけ or 命令（「〜してみろ」「〜を忘れるな」）
   ```

3. **ランキング型の場合**：下位から始めて、1位で最大のインパクトを出す
4. **リスト型の場合**：最も強い項目を1番目か最後に置く（初頭効果 or 新近性効果）

**本文中の興味維持テクニック：**

| テクニック | やり方 | 効果 |
|-----------|--------|------|
| 太字アンカー | 数字とキーワードを太字に | スキャンリーディングでも要点が分かる |
| 会話調ツッコミ | 「〜ってことなw」「（私は見たことない）」 | 親近感＋離脱防止 |
| アナロジー | 「パンデミック直前」「ゴールドラッシュ」 | 複雑な概念を一瞬で理解 |
| 時系列展開 | 2022年→2023年→2024年→… | 「加速感」を体感。「次は？」で読み続ける |
| 対話形式 | 「今日はいい天気ですね」「ですね(YES)」 | 読者を会話の当事者にする |
| 具体的な金額 | 「78,000円」「150万」「$100」 | お金の数字は絶対に目が止まる |

**本文中の画像挿入ルール：**

画像はただの「飾り」ではなく、戦略的に配置する装置。以下のタイミングで挿入する。

1. **感情増幅ポイント**：恐怖や興奮を煽るテキストの直後
   - 例：「パンデミック直前と同じ」の後に→ 巨人が街を蹂躙するマンガコマ
   - 効果：テキストで煽った恐怖を視覚で追い打ち。感情のピークを作る

2. **概念の可視化ポイント**：抽象的な概念を説明した直後
   - 例：「旧人類と新人類は分断された」の後に→ 猿人から新人類への進化イラスト
   - 効果：テキストを読まなくても一枚で概念が伝わる

3. **証拠提示ポイント**：「本当にやった」を証明すべき箇所
   - 例：「14タスクを処理した」の後に→ Coworkの実際の操作画面スクリーンショット
   - 効果：主張の信頼性を視覚的に裏付ける

4. **Before/Afterポイント**：変化を見せたい箇所
   - 例：「手書きノートからスライドを生成」→ 手書きノート写真＋完成スライド画像
   - 効果：変換の衝撃を一目で理解させる

**画像のキャプション**も重要：
- 「Coworkが自動でXを検索し、バズポストを分析している様子」のように、画像が何を示しているかを説明する
- キャプションは画像の直下に配置し、テキストの流れを途切れさせない

### パート6：まとめ（哲学的昇華）

リスト形式の本文を「一段上の視座」で締める。

**書き方：**
- 本文の具体論を抽象化し、人生哲学レベルに昇華させる
- 読者の心に残る「一言」を入れる

**実例：**
- 「心理学って、結局は『脳のバリア』をどう外すか、なんです」
- 「給料を上げるというのは、自分という商品の価値を、市場という名のオークションにかけ続ける行為です」
- 「ゴールドラッシュで一番儲けたのは、金を探した奴じゃなく『シャベルを売った奴』」

**構造：**
```
「最後に。」or 「結論：」
→ 本文のエッセンスを1〜2文に凝縮
→ 読者への最後の問いかけ or 宣言
```

### パート7：6教育⑤⑥の組み込み＋行動喚起への橋渡し

まとめとCTAの間に、残り2つの教育を自然に埋め込む。

**⑤ 投資の教育**（行動にはコストが必要。でもリターンは大きい）
- 「Maxプランのたった$100で、月160万円分の作業が完了する」
- 「セットアップだけで数十万〜百万以上請求できるブルーオーシャン」
- 投資対効果を数字で示すことが重要

**⑥ 行動の教育**（今すぐ第一歩を踏み出せ）
- 「この流れに乗るか、AIに仕事を奪われる『その他大勢』として淘汰されるか」
- 二択を提示し、「行動しない」選択肢をコストの高いものとして描く
- 直後のCTAへの心理的準備を整える

### パート8：CTA（Call To Action）

記事の最終目的。ここまでの全文が「CTAを押させるための伏線」。

**CTA構造テンプレート：**

```
[ブリッジ文（記事の価値を一言で要約し、「もっと知りたいなら」と繋ぐ）]

[主CTA（フォロー / LINE登録 / コミュニティ参加 etc.）]

[副CTA（いいね / リポスト / 引用RT）]

[次回予告 or 関連記事リンク]
```

**CTA 5つの型：**

**型1：恐怖FOMO型（攻撃型向き）**
```
取り残されたくないやつは、いいねとフォロー必須→ @xxx
引用感想も待ってる。
次の記事もお楽しみに！
```

**型2：価値提供継続型（知的メンター型向き）**
```
……とまあ、こんな感じで、「知っているだけで得をする、知らないと搾取される」
そんなビジネスパーソンに有益な記事を他にもいろいろ書いてます。
もっと深い話が気になる人は、フォローしておいてくださいね。
次は、何をハックしましょうか。
```

**型3：コミュニティ誘導型**
```
[ツール名/テーマ]に興味ある人
ここから↓
オープンチャット「[名前]」
[URL]
```

**型4：追記型（本業誘導）**
```
追記
最後に、私の本業の話を少し。
[本業の紹介]
現在、このX経由でご連絡いただいた方限定で[特典]のキャンペーンをやっています。
プロフィールのリンクからどうぞ。
```

**型5：軽量エンゲージメント型**
```
もし、こういう「[記事のテーマ]」をフィードに流したいなら、俺（@xxx）をフォローしておけ。
あと、既読感覚で「いいね」と「リポスト」よろ🫰
```

**CTAのチェックリスト：**
- [ ] 行動のハードルが最小か？（「既読感覚で」「フォローしておいて」）
- [ ] 恐怖 or 希望で動機づけしているか？
- [ ] 次回予告 or 関連記事で継続接触の仕掛けがあるか？
- [ ] 本文全体がCTAの根拠になっているか？

### パート9：関連記事リンク

CTAの後に、過去記事を2〜3本埋め込む。

**効果：**
- プロフィールに飛ばなくても過去記事に触れられる
- 「この人、他にも良い記事書いてるんだ」で信頼感アップ
- 滞在時間・接触回数の増加→フォロー率アップ

**選び方：**
- テーマが近い記事（「心理学の記事を読んだ人は、説得術の記事も読みたい」）
- エンゲージメントが高い記事（いいね数が見えるので社会的証明になる）

## 文体ガイドライン（3タイプ）

### タイプA：攻撃型（煽り・上から目線）

**語尾：** 「〜だ」「〜しろ」「〜を忘れるな」「〜な奴は終わってる」
**二人称：** 「お前」「奴ら」「アホ」
**特徴的表現：**
- 「〜してる奴は一生三流」
- 「〜の準備はできてるか？」
- 「〜を叩き込め」
- 下ネタやスラングを適度に（記憶に残りやすい）
- 自虐を混ぜて親近感（「ニートみたいに」「ポテチ食いながら」）

**注意：** 攻撃は読者の「行動」に向ける。人格攻撃は不快なだけ。「お前がバカ」ではなく「お前のやり方がバカ」。

### タイプB：知的メンター型（丁寧・知的）

**語尾：** 「〜です」「〜ですよね」「〜なんです」
**二人称：** 「あなた」「みなさん」（省略が多い）
**特徴的表現：**
- 「実は、〜なんです」（秘密を教える口調）
- 「悪用厳禁ですよ」（ユーモアを含んだ注意）
- 「〜ありませんか。」（共感の問いかけ）
- 哲学的な一言で締める

**注意：** 丁寧すぎると退屈。要所で断定（「これが真実です」）を入れてメリハリ。

### タイプC：体験共有型（友達の報告）

**語尾：** 「〜した」「〜だった」「〜してくれた」
**二人称：** 「あなた」（使う場面を絞る）
**特徴的表現：**
- 「正直に言う。」（冒頭の宣言）
- 数字と事実の列挙
- 「〜してくれた」（ツールを擬人化）
- 「別に自慢したいわけじゃない」（謙遜で好感度UP）

**注意：** 事実の羅列だけだと単調。感想（「エグくない？」）を適度に挟む。

## 画像設計ガイド

記事内の画像は「装飾」ではなく「戦略的な心理装置」。以下の原則で配置する。

### ヘッダー画像の設計原則

1. **ダークモードTLで映える色調**を使う（X利用者の大半がダークモード）
2. **画像内にタイトル文字を入れる**（画像だけで記事内容が分かるように）
3. **ブランド統一**（毎回同じテイストで、TLで「あの人の記事だ」と瞬時に認識させる）

### 本文中の画像挿入タイミング

| 挿入タイミング | 目的 | 例 |
|--------------|------|-----|
| 恐怖・興奮テキストの直後 | 感情増幅 | 「パンデミック直前」→ 巨人が街を蹂躙するマンガコマ |
| 抽象概念の説明直後 | 概念の可視化 | 「旧人類と新人類の分断」→ 進化のイラスト |
| 「本当にやった」主張の直後 | 証拠提示 | 「14タスク処理」→ 操作画面のスクリーンショット |
| 変化を見せたい箇所 | Before/After | 手書きノート → 完成スライド |

### キャプションの書き方
- 画像の直下に配置
- 「何が映っているか」+「だから何なのか」を1文で
- 例：「Coworkが自動でXを検索し、バズポストを分析している様子。いいね数、テーマ、伸びた理由まで全自動で記録する。」

## 「AI臭さ」を消すための文体ルール

バズ記事で最も致命的なのは「AI臭い」文章。以下を守ること。

1. **同じ文末を3回連続で使わない**（「〜です。〜です。〜です。」は最悪）
2. **体言止めを適度に使う**（「それが、説得術。」「究極のエンドゲーム。」）
3. **口語表現を混ぜる**（「マジで」「ぶっちゃけ」「〜なんだわ」）
4. **不完全な文を入れる**（「信用の面積が広がれば、そこに乗っかる『お金』の額は、後から勝手についてきます。」）
5. **読点の位置を不規則に**（機械的に等間隔で打たない）
6. **括弧内のツッコミ**（「（私は見たことない）」「（断りづらい……）」）
7. **感嘆詞・擬態語**（「ぶちまける」「ぶっ刺さる」「ゴリゴリに」）
8. **wやスラング**（文体タイプに合わせて適切に。攻撃型は多め、知的型は控えめ）
