# x-buzz-article — Xバズ記事作成スキル

## 概要

X（Twitter）のArticles機能を使ったバズる長文記事を、構造的に設計して作成するClaude Cowork用スキル。

実際に**337万imp / いいね2,679**、**137万imp / いいね508**、**63万imp / いいね1,019**を記録した記事群を徹底分析し、そのバズの法則をすべてスキルに組み込んでいる。

## ファイル構成

```
x-buzz-article/
├── SKILL.md        ← スキル本体（Claude Coworkが読み込む指示書）
├── analysis.md     ← 5記事の徹底分析レポート（8章構成）
└── README.md       ← この説明書
```

## スキルの機能

### 9パート構造テンプレート

記事を以下の9パートで構成し、読者を最後のCTAまで離脱させない設計にする。

1. **タイトル** — 【隅付き括弧】+数字+ベネフィットの組み合わせ（5パターン）
2. **ヘッダー画像** — ダーク/マンガ/ミニマルの3スタイル
3. **フック（冒頭3行）** — 恐怖/共感/欲望/事実提示の4パターン
4. **問題の提起** — 6教育①②③（目的・信頼・問題）の同時組み込み
5. **本文（リスト型セクション）** — 6教育④（手段）+ 興味維持テクニック6種
6. **まとめ（哲学的昇華）** — 具体論を人生哲学レベルに抽象化
7. **6教育⑤⑥の組み込み** — 投資の教育＋行動の教育
8. **CTA** — 5つの型（FOMO型/価値提供型/コミュニティ型/追記型/軽量型）
9. **関連記事リンク** — 過去記事の埋め込みで接触回数UP

### 3つの文体タイプ

| タイプ | 特徴 | モデル |
|-------|------|-------|
| 攻撃型 | タメ口・煽り・命令形 | @proneeeet風 |
| 知的メンター型 | 丁寧語ベース・知的 | @fudoosanya風 |
| 体験共有型 | 過去形主体・友達の報告 | @oda_nobunaga10風 |

### 6教育チェックリスト

記事完成後に6つの教育（目的・信頼・問題・手段・投資・行動）がすべて含まれているか確認する機能付き。

### 画像設計ガイド

画像を「装飾」ではなく「戦略的な心理装置」として配置するためのガイド。感情増幅・概念可視化・証拠提示・Before/Afterの4タイミングで挿入。

### AI臭さを消す8つのルール

テンプレ的な文章にならないための具体的なルール（同じ文末の禁止、体言止め、口語表現、括弧ツッコミなど）。

## インストール方法

### Claude Coworkの場合

1. `x-buzz-article.skill` ファイルをダウンロード
2. Coworkの画面で「コピーしてスキルに追加」をクリック

### 手動インストール

1. `SKILL.md` を以下のパスに配置：
   ```
   ~/.claude/skills/x-buzz-article/SKILL.md
   ```

## 使い方

Coworkに以下のように話しかけるだけでスキルが起動する：

- 「X記事を書いて」
- 「バズ記事を作って」
- 「セールスレター型の記事を書きたい」
- 「タイトルを考えて」
- 「フックを書いて」
- 「CTAを作って」
- 「6教育を入れて」

スキルが起動すると、テーマ・ターゲット読者・文体タイプ・CTA先・画像の方向性・リスト数を確認してから記事を生成する。

## analysis.md（分析レポート）の内容

8章構成で各記事を徹底分析：

| 章 | 内容 |
|----|------|
| 第1章 | タイトルの構造分析（5パターン＋法則5つ） |
| 第2章 | フックの構造分析（4パターン＋法則5つ） |
| 第3章 | 本文中の興味付けテクニック（7種＋法則6つ） |
| 第4章 | 6教育の組み込み分析（各記事ごとに6要素を特定） |
| 第5章 | CTAの構造分析（5つの型＋法則6つ） |
| 第6章 | 画像の訴求分析（ヘッダー＋本文中＋法則6つ） |
| 第7章 | 文体・トーン分析（3パターン＋法則3つ） |
| 第8章 | 構造テンプレート（全記事共通パターン） |
