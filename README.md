# X自動運用システム — Claude Cowork

@oda_nobunaga10 のX(Twitter)運用をClaude Coworkで半自動化するためのシステム。タイムラインの調査、引用リポストの作成、投稿までをChromeブラウザ操作で一気通貫で実行する。

---

## できること

- タイムライン上のバズポストを自動スキャンし、AI/テック/ビジネス系で高エンゲージメントのポストを選定する
- 140文字ぴったりの引用リポストを @oda_nobunaga10 の口調で生成する
- Chromeブラウザを操作してXの引用リポスト機能から直接投稿する
- 指定ポストに対する連結リプライ（4〜5本）を調査ベースで作成・投稿する

---

## 運用ルール

### 引用リポスト

| 項目 | ルール |
|---|---|
| 文字数 | 140文字ぴったり（JavaScriptで毎回検証） |
| 口調 | 「ガチ」「エグい」「〜だろ」「〜だよな」 |
| 絵文字 | 使わない |
| 断定口調 | 「〜かも」「〜と思う」は禁止。言い切る |
| 禁止接続詞 | 「しかし」「さらに」「つまり」「また」「一方で」「で、」 |
| 許可接続詞 | 「けど」「ただ」「要は」 |
| 投稿方法 | X標準の引用機能を使用。RTのみは禁止 |

### ポスト選定基準

| 条件 | 基準 |
|---|---|
| エンゲージメント | いいね100以上を目安 |
| ジャンル | AI / テック / ビジネス関連 |
| 鮮度 | 直近数時間以内 |

### 投稿の構造パターン

```
[感情フック] → [事実の再解釈] → [行動喚起]
```

冒頭で感情的に引き込み、元ポストの要点を自分の言葉で再構成し、行動を促して締める。

## 使い方

### 引用リポストを投稿する

Coworkに以下のように指示する。

```
タイムラインで伸びてるポストを見つけて引用リポストを作成・投稿して
```

特定のポストを引用したい場合はURLを指定する。

```
引用リポスト作成
https://x.com/username/status/1234567890
```

### 連結リプライを投稿する

```
「Claude Code」でXリサーチして、このポストに連結リプ作って
https://x.com/username/status/1234567890
```

## 凍結リスクについて

このシステムはChromeブラウザの通常操作を再現しているだけで、APIの大量リクエストや自動化ツールによる不正アクセスは行っていない。安全に運用するために以下を守る。

- 投稿間隔をあける（1時間に数件程度）
- 引用文の内容を毎回変える
- いいね・フォローの連打をしない

---

## プロンプト設計7原則

この運用から抽出した、Claude活用全般に転用可能な設計原則。

1. **自由度を下げれば品質が上がる** — 「140文字ぴったり」で推敲を強制
2. **禁止リストは許可リストより強力** — 「しかし」を禁止すると口語に自動シフト
3. **口調は語彙リストで定義する** — 「カジュアルに」ではなく具体的な4語を指定
4. **構造を固定し表面を変える** — [感情フック → 再解釈 → 行動喚起]
5. **機械的検証を組み込む** — JavaScriptで文字数を毎回検証
6. **選定基準を数値化する** — 「いいね100以上」「直近数時間」
7. **フィードバックを禁止リストに変換する** — 運用するほど品質が上がる仕組み

詳細な分析は `X運用_品質分析_プロンプト設計の解剖.md` を参照。
